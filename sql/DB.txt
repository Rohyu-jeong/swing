DataBase
- 데이터들의 조직화된 모음
- 컴퓨터에 전자적으로 저장이 되고 데이터 베이스 관리 시스템에 의해 제어가 된다.

DBMS(Data Base Management System)
- 데이터베이스와 사용자 또는 프로그램 간의 매개체 역할을 하여 사용자가 정보의 구성, 검색, 수정, 삭제와 같은 관리를 할 수 있게 해준다.

DBMS의 종류
계층형 데이터베이스(HDBMS)
- 폴더와 파일 등의 계층 구조로 데이터를 저장하는 방식
- 하드디스크, DVD 파일 시스템 등
- 데이터 접근 속도가 빠름
- 데이터의 사용량 예측이 쉬움
- 상하 종속적인 관계로 구성되어 초기 세팅 후 프로세스 수용이 어려움

네트워크형 데이터베이스(NDBMS)
- 데이터 구조를 네트워크 상의 노드 형태로 논리적이게 표현한 데이터 모델
- 계층형 데이터베이스의 데이터 중복 문제 해결

관계형 데이터베이스(RDBMS)
- 행과 열을 가지는 표 형식 데이터를 저장하는 형태의 데이터베이스
- SQL을 이용하여 관리 및 접근
- 다양한 용도(범용성), 높은 성능
- 데이터의 분류, 정렬, 탐색 속도가 빠름
- 관계를 통해 각 데이터를 중복없이 한 번만 저장

인기있는 RDBMS 소프트웨어
- My SQL
- Microsoft SQL Server
- PostgreSQL
- Oracle
- DB2


계정

관리자 계정(Administrator)
- 데이터베이스의 생성과 관리를 담당하는 슈퍼유저의 계정이며, 정의, 생성, 삭제 등의 작업이 가능하다.
- 오라클 데이터베이스 생성시 관리자 계정인 SYS와 SYSTEM 계정이 자동으로 생성된다.
- 데이터베이스에 대한 모든 권한과 책임을 가지는 계정이다.

일반 계정
- 데이터베이스에 대하여 명령, 갱신, 보고서 작성 등의 작업을 수행할 수 있는 계정
- 일반 계정은 업무에 필요한 최소한의 권한만 가지는 것을 원칙으로 한다.

계정 잠금 풀기
alter user 계정명 account unlock;

비밀번호 변경하기
alter user 계정명 identified by 비밀번호;

스키마
1. 데이터베이스의 구조와 제약 조건에 관한 전반적인 명세를 기술한 메타데이터의 집합
2. 사용자의 관점에 따라 외부 스키마, 개념 스키마, 내부 스키마로 나뉜다.

스키마의 종류
외부스키마
- 사용자 개인의 입장에서 필요한 데이터베이스의 논리적 구조를 정의
- 실제 세계에 존재하는 데이터들을 어떤 형식, 구조, 배치, 화면을 통해 사용자에게 보여줄 것인가.
- 하나의 데이터베이스에는 여러 개의 외부스키마가 존재할 수 있다.
- 개발자는 C, Java 등의 프로그래밍 언어로 DB에 접근한다.

개념스키마
- 조직체 전체를 관장하는 입장에서 DB를 정의한 것
- 관계, 제약조건, 접근 권한, 보안 정책, 무결성 규칙에 관한 사항을 포함하고 있다.

내부스키마
- 스키마를 디스크 기억장치에 물리적으로 구현하기 위한 방법을 기술한 것
- 데이터베이스의 물리적 저장 구조를 정의
- 디스크에 어떤 구조로 저장할 것인가
- 데이터의 실제 저장 방법을 기술
- 물리적 저장 장치와 밀접한 계층

테이블 = 릴레이션
- 행과 열로 이루어진 데이터의 집합(엑셀의 모습과 흡사)
- 일반적인 데이터베이스에서는 행과 열만 있으면 테이블이라고 하지만, 관계형 데이터베이스에서는 특별한 제약을 추가해서 릴레이션이라고도 부른다.

행(row)
- 테이블을 구성하는 데이터들 중 가로로 묶은 데이터 셋을 의미
- 일반적으로 한 명에 대한 정보를 가지고 있다.
- 관계형 데이터베이스에서는 튜플(tuple), 또는 레코드(record)라는 이름으로 불린다.
- 릴레이션의 튜플의 개수를 카디널리티

열(column)
- 테이블을 구성하는 데이터들 중 세로로 묶은 데이터셋
- 일반적으로 열은 그 테이블의 속성을 의미한다.
- 열을 구성하는 값들은 같은 도메인으로 되어있다.
- 관계형 데이터베이스에서는 속성(Attribute)라고 한다.
- 속성의 개수를 차수(degree)라고 한다.

도메인(domain)
- 하나의 속성이 취할 수 있는 동일한 유형의 데이터 집합

SQL문
- 원하는 결과를 얻어오기 위해 DB에 요청하는 문장
- SQL 문장은 대소문자를 구별하지 않는다.
- 한 줄 또는 여러 줄에 걸쳐 입력하는 것이 가능
- SQL 문장의 끝은 세미콜론(;)으로 맺어야 한다.

SQL 문장의 종류
1. DDL(Data Definition Language) : 데이터 정의어
- CREATE, ALTER, DROP, TRUNCATE 

2. DML(Data Manipulation Language) : 데이터 조작어
- SELECT, INSERT, UPDATE, DELETE

3. DCL(Data Control Language) : 데이터 제어어
-  GRANT, REVOKE

4. TCL(Transaction Control Language) : 트랜잭션 제어어
- COMMIT, ROLLBACK, SAVEPOINT

DDL
- 테이블 조작, 제어 관련 SQL
- CREATE : 테이블 생성 관련
- DROP : 테이블 삭제(복구 안됨)
- ALTER : 수정
   - 테이블 명 수정 : ALTER TABLE [원래 이름] RENAME TO [새로운 테이블명]
   - 속성 추가 : ALTER TABLE [테이블 명] ADD([새로운 속성명] [자료형])
   - 속성명 수정 : ALTER TABLE [테이블 명] RENAME COLUMN [속성명] TO [새로운 속성명]
   - 속성 삭제 :  ALTER TABLE [테이블 명] DROP COLUMN [생성된 속성명]

제약조건
테이블에 문제가 되는 결함이 있는 데이터가 입력되지 않도록 미리 지정해둔 조건
테이블을 생성할 때 함께 설정할 수 있고, 추후에 생성하거나 변결할 수도 있습니다.

이미 데이터가 포함되어 있는 상황에서 제약조건을 함부로 바꾸기 어려우므로 사전에 테이블 설계를 잘 해두는 편이 좋다.

PRIMARY KEY (고유키)
데이터가 NULL 값일 수 없고 중복되서도 안된다.

FOREIGN KEY (외래키)
다른 테이블 열을 참조하여 해당 테이블에 존재하는 값만 입력 가능

UNIQUE (유일키)
NULL 값은 가능하나 중복되는 값은 들어갈 수 없는 제약조건

NOT NULL
NULL 값을 불가 하지만 중복은 가능

DEFAULT
아무것도 입력하지 않으면 설정한 기본값이 입력됨
값이 입력되면 기본값은 무시됨

CHECK
설정한 조건식을 만족하는 데이터만 입력 가능
조건식을 만족하지 않는 데이터는 입력이 거부

자료형
- 컬럼(속성)에 저장되는 데이터의 유형을 말한다.

문자형 자료형
CHAR(n)	고정길이 문자

VARCHAR2(N)	가변길이 문자(4000BYTE)

CLOB	대용량 텍스트 데이터 타입 (4GB 이상)

숫자 데이터 타입
NUMBER(P, S) 가변 숫자 P : 자리수 (1 ~ 38) / S : 소수점자리수(-84 ~ 127)
FLOT(P) : NUMBER의 하위 타입 P : 소수점 자리수(1 ~ 128)

날짜 데이터 타입
DATE : BC4712년 1월 1일부터 9999년 12월 31일, 연, 월, 일, 시, 분, 초까지 입력 가능
TIMESTAMP : 연도, 월, 일, 시, 분, 초, 밀리초까지 입력가능

테이블 생성하는 법
CREATE TABLE 테이블명 (
	컬럼명 타입(길이),
	컬럼명 타입(길이),
	컬럼명 타입(길이)
);
